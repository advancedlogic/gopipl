# PIPL Desktop

A cross-platform desktop client for PIPL, built with [Wails v2](https://wails.io)
(Go) and [Svelte 5](https://svelte.dev) (runes). It is a third front end
alongside the flag CLI (`cmd/pipl`) and the Bubble Tea TUI (`internal/tui`),
and — like them — drives `internal/chat` for **everything security-relevant**.

The Go/JS boundary carries only display projections. No key material,
plaintext, or capability ever crosses into the frontend: `app.go` forwards
each call to the engine, which keeps all crypto on the client and never sends
a key to the server. This preserves the invariant in the repo's `CLAUDE.md`
that the three front ends cannot drift.

## What it does

- **First-run setup** — creates this device's identity (handle + optional
  coordination server) via `chat.Init`, or loads the existing `~/.pipl` state.
  It is the *same peer* the CLI uses (honors `$PIPL_HOME`).
- **Conversation list** — folder- and relay-backed conversations, ordered by
  activity, showing the transport and last message.
- **Message thread** — decrypted history with day separators; own messages
  show their known audience for per-recipient sends.
- **Composer with recipient selection** — a Recipients panel toggles who each
  message goes to. The UI names the audience model in force (GROUP shared key
  vs. SEPARATE per-recipient keys) exactly as the CLI/TUI do, and refuses to
  send to nobody.
- **Revocation** — per-message actions: hard-revoke a recipient of a separate
  send, hide/unhide from everyone, or delete permanently. Each surfaces the
  honest-limitation notice ("revocation cannot un-share").
- **New / Join** — create a relay or folder conversation and copy the invite
  code; join by invite code or shared folder.
- **TOFU notices** — first-contact fingerprints are surfaced for out-of-band
  verification.
- **Live updates** — the app subscribes to backend change events so the list
  and thread refresh without manual reload.

## Layout

```
main.go                 Wails app entry (window, embed, bind)
app.go                  the single Wails binding -> internal/chat (DTOs only)
frontend/
  src/App.svelte        router: splash / setup / shell
  src/lib/api.js        wrapper over the generated wailsjs bindings
  src/lib/store.svelte.js   reactive store (runes) + event wiring
  src/components/       Sidebar, Conversation, composer, modals, actions
  wailsjs/              generated bindings (regenerated by wails, gitignored)
```

## Develop

```powershell
cd desktop
wails dev            # hot-reload frontend + Go, live app window
```

## Build

```powershell
cd desktop
wails build          # -> build/bin/pipl-desktop.exe (or .app / binary)
```

Prereqs: the Wails CLI (`go install github.com/wailsapp/wails/v2/cmd/wails@latest`),
Node, and Go 1.24+. On Windows the WebView2 runtime is required (bundled on
current Windows 11).

## Notes

- The `desktop/` module has its own `go.mod` with a `replace` pointing at the
  parent, so the engine, identity, and crypto packages are reused verbatim.
- `wailsjs/` is generated (`wails generate module`) and gitignored; any
  `wails build`/`wails dev` recreates it from `app.go`.
- Running several peers on one machine: launch with a different `PIPL_HOME`
  per window, just like `pipl -home`.
